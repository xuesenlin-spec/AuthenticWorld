## Helper 函数鲁棒性分析

### `_check_calendar_has_event`
- **实现**: 通过 `env.controller.pull_file()` 拉取 Calendar DB 到本地，用 Python `sqlite3` 查询
- **异常处理**: 完整 `try/except Exception: return False` 包裹
- **状态**: ✅ 安全

### `_check_file_contains`
- **实现**: 检查文件是否存在 → `cat` 读取内容 → 检查子串匹配 → 回退到模糊文件名匹配
- **异常处理**: 无显式 try/except，但依赖 adb 输出解析而非异常
- **状态**: ✅ 安全（文件不存在时返回 False）

### `_check_sms_sent`
- **实现**: 查询 `content://sms/sent`，检查号码和 fuzzy match 消息体
- **异常处理**: `_get_sent_messages` 处理空结果，`was_sent` 处理解析错误
- **状态**: ✅ 安全

### `_get_android_time`
- **实现**: `adb shell date +%s` 获取时间戳
- **异常处理**: 无显式 try/except
- **状态**: ⚠️ 轻微风险（adb 失败时 `int()` 会报错，但概率极低）

## 发现的关键问题

### 系统性 Bug：`simple contacts pro` 包名映射错误（影响 12/20 任务）

**位置**: `pega.py` 第 48 行

```python
"simple contacts pro": "com.android.contacts",   # ❌ 错误
```

**正确值**:
```python
"simple contacts pro": "com.simplemobiletools.contacts.pro",   # ✅ 正确
```

**影响**:
- `_clear_task_app_data` 会尝试清理 `com.android.contacts`（可能未安装）
- 实际的 `com.simplemobiletools.contacts.pro` 数据不会被清理
- 联系人数据会在任务间泄漏，可能导致误判

**受影响的任务**: 3, 4, 9, 11, 13, 14, 15, 16, 17, 18, 19, 20
